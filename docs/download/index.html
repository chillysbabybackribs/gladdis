What Gladys actually does

A trusted local app, not a sandboxed assistant. The agent can act on your behalf with the same reach you have as the OS user.

Split workshop layout

Conversation on the left, a native multi-tab Chromium browser on the right. Each tab is an Electron WebContentsView rendered as a real OS layer, not an iframe.

Real browser automation

The agent navigates, clicks, types, and presses keys in the live browser — discovering targets and acting on them as one step, with trusted input dispatched at real coordinates.

Full Chrome DevTools Protocol

Every tab attaches the CDP debugger with DOM, network, console, runtime, storage, and performance domains enabled — reachable through structured calls or a raw command escape hatch.

Filesystem and shell

Read, write, and surgically edit files, list directories, search recursively, and run arbitrary shell commands across everything the OS user can reach.

Repository intelligence

Workspace orientation, task-shaped grep, bounded span reads, and validation gates let the agent understand a codebase before it edits, then verify the change.

Custom agents and memory

Save reusable agent blueprints with model preferences, tool constraints, and workflows. Working memory and history recall keep long sessions coherent without bloating the prompt.

Web research in-tab

Search, deep recursive crawl, and single-URL fetch all surface their results in the visible Chromium tab, so research stays transparent and inspectable.

Local terminal

A real PTY-backed terminal lives inside the app for installs, builds, tests, and dev servers — alongside the chat and browser, never in a separate window.

The core design bet

Perception is grep and accessibility, not pixels

Instead of guessing from screenshots, Gladys reads the live page the way the page is actually built. A grep engine runs inside the page and returns matched elements with a stable selector, visibility, and live bounding-box coordinates. An accessibility-tree snapshot exposes roles, names, states, and stable refs for component-heavy UIs.

That makes targeting more precise than vision — a literal node and a literal coordinate, not pixels the model has to infer. Screenshots stay available as a fallback for genuinely visual content.

  • grep_page Find exact text, regex, or selectors on the live page with surrounding context and coordinates.
  • read_a11y Capture a compact accessibility tree with stable refs for reliable control discovery.
  • grep_click / grep_type Discover and act in a single primitive, dispatched at real coordinates.
  • watch_network Read the JSON behind the render — the data a page loads, not just its markup.

Bring your own models

Completions run in the main process, so API keys never reach the renderer. Keys are stored encrypted on-device through the OS keychain.
